Lukardor Music einrichten

Damit alle Songs, Favoriten und Playlists sehen, braucht die App einen kostenlosen Online-Speicher. Einmalig ca. 5 Minuten.

1

Supabase-Projekt erstellen

Auf supabase.com kostenloses Konto + neues Projekt anlegen.

2

Schlüssel eintragen

Unter Project Settings → API die Project URL und den publishable/anon Key kopieren und oben in dieser Datei eintragen.

3

Anmeldung, Datenbank & Rechte anlegen

Aktiviere zuerst unter Authentication → Sign In / Providers die anonyme Anmeldung (damit jedes Gerät eine Identität für Besitzer-Rechte bekommt). Dann den Code im SQL Editor einfügen und Run klicken.

create table if not exists songs (
  id text primary key,
  title text, artist text, album text,
  duration float8, audio_path text, cover_path text,
  owner_id uuid default auth.uid(),
  created_at timestamptz default now()
);
alter table songs enable row level security;
create policy "songs read"   on songs for select using (true);
create policy "songs insert" on songs for insert to authenticated with check (auth.uid() = owner_id);
create policy "songs update" on songs for update to authenticated using (auth.uid() = owner_id);
create policy "songs delete" on songs for delete to authenticated using (auth.uid() = owner_id);

create table if not exists favorites (
  user_id uuid default auth.uid(),
  song_id text references songs(id) on delete cascade,
  created_at timestamptz default now(),
  primary key (user_id, song_id)
);
alter table favorites enable row level security;
create policy "fav read"   on favorites for select to authenticated using (auth.uid() = user_id);
create policy "fav insert" on favorites for insert to authenticated with check (auth.uid() = user_id);
create policy "fav delete" on favorites for delete to authenticated using (auth.uid() = user_id);

create table if not exists playlists (
  id text primary key, name text,
  created_at timestamptz default now()
);
alter table playlists enable row level security;
create policy "pl read"   on playlists for select using (true);
create policy "pl insert" on playlists for insert with check (true);
create policy "pl update" on playlists for update using (true);
create policy "pl delete" on playlists for delete using (true);

create table if not exists playlist_songs (
  playlist_id text references playlists(id) on delete cascade,
  song_id text references songs(id) on delete cascade,
  added_at timestamptz default now(),
  primary key (playlist_id, song_id)
);
alter table playlist_songs enable row level security;
create policy "pls read"   on playlist_songs for select using (true);
create policy "pls insert" on playlist_songs for insert with check (true);
create policy "pls delete" on playlist_songs for delete using (true);

insert into storage.buckets (id, name, public)
values ('music','music', true) on conflict (id) do nothing;
create policy "music read"   on storage.objects for select using (bucket_id='music');
create policy "music upload" on storage.objects for insert with check (bucket_id='music');
create policy "music delete" on storage.objects for delete to authenticated using (bucket_id='music' and owner = auth.uid());
4

Speichern & teilen

Datei speichern, neu laden, an Freunde weitergeben.

Jeder mit der Datei kann hochladen/löschen. Datei-Limit (Gratis): 50 MB pro Song, 1 GB gesamt.
